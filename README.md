# Pro Store Kit
Stack frontend modular empresarial orientada a identidad corporativa y gestión de catálogos complejos.

## 🚀 Descripción del Proyecto
**Pro Store Kit** es un ecosistema de componentes y vistas diseñado para resolver la complejidad visual y funcional del comercio digital moderno. Se centra en una arquitectura de componentes desacoplados que permite una integración segura con cualquier API, manteniendo una estética limpia y profesional.

## 🎯 Propósito del Sistema
Este kit ha sido concebido como una base de alto nivel para aplicaciones empresariales que requieren una **identidad corporativa sólida** y una gestión de datos compleja. El objetivo es reducir el tiempo de salida al mercado proporcionando una arquitectura pre-configurada que cumple con los estándares más exigentes de seguridad y usabilidad.

## 🛠️ Stack Tecnológico
* **Core:** Vue.js 3 (Composition API).
* **UI Components:** PrimeVue (Librería de componentes de alta calidad).
* **Icons:** Material Icons (Google Material Design).
* **Language:** TypeScript (Tipado estricto para modelos y estados).
* **State Management:** Pinia (Gestión de estado modular y segura).
* **Networking:** Axios (Cliente HTTP para consumo de APIs).
* **Internationalization:** vue-i18n (Soporte multi-idioma nativo).
* **Tooling:** Vite (Entorno de desarrollo rápido y optimizado).

## 📦 Módulos Integrados
* **Gestión de Catálogo:** Caracterización profunda de productos con soporte para variantes técnicas y atributos dinámicos.
* **Identidad y Acceso:** Módulos de autenticación y perfiles orientados a entornos corporativos.
* **Sistema de Adjuntos:** Gestión centralizada de archivos con previsualización integrada.
* **Gestión de Permisos:** Control de acceso basado en roles (RBAC) plenamente tipado.

---

## 🏗️ Estructura de Arquitectura
1.  **Types/Interfaces:** Contratos de datos tipados para todo el flujo de información.
2.  **Stores (Pinia):** Gestión de estado reactivo y persistente por módulos (auth, cart, catalog).
3.  **Services (Axios):** Capa de abstracción para el consumo de APIs externas con interceptores de seguridad.
4.  **Composables:** Lógica de negocio extraída y plenamente tipada.
